# NoteSeed MVP 开发任务书（Development Plan）

> **目标**：指导 AI 编码代理（Claude Code / Cursor 等）连续完成 NoteSeed v1.0 MVP 的完整开发。
>
> **范围**：PRD v1.0 定义的 MVP 能力（PageSense / Contextualizer-简版 / Distiller / Tagger / Cardwright / Dispatcher 至 Memos + 飞书 / Chronicle-简版）。
>
> **读者**：AI 编码代理。每个任务都包含「目标 / 输入 / 产出 / 验收 / 依赖」，可直接作为 AI 的任务 prompt。
>
> **文档版本**：v1.0 · 最后更新 2026-04-16

---

## 如何使用本文档

### 给 AI 编码代理

- 按 **Phase 顺序执行**，每个 Phase 完成后运行 `pnpm test` 和「验收清单」确认通过再进入下一 Phase
- 每个任务都有明确的**输入文件**和**产出文件**，不要擅自扩大修改范围
- 遇到模糊决策，优先参考本文档「默认决策表」，其次遵循 PRD，最后才能自行判断
- 每完成一个任务，在任务末尾写一份 `DONE.md` 进度日志，记录改动文件列表和潜在风险

### 给执行者（人类）

- 每个 Phase 完成后做一次 code review
- Phase 之间是**强隔离**的，上一 Phase 不通过不要开下一 Phase
- 遇到 AI 产出不符合预期时，引用本文档的「验收清单」反馈，而非口头描述

## 全局约定

### 技术栈（不可更改）

| 层 | 技术 |
|---|---|
| 前端（插件） | React 18 + TypeScript 5 + Vite + CRXJS + Tailwind CSS 3 + shadcn/ui + Zustand |
| 后端 | Node.js 20 + Fastify 4 + TypeScript 5 + Prisma + Zod |
| 数据库 | PostgreSQL 16 + Redis 7 |
| AI | Anthropic Claude API（Haiku + Sonnet） |
| 包管理 | pnpm（workspace monorepo） |
| 测试 | Vitest + Playwright |

### 仓库结构（monorepo）

```
noteseed/
├── apps/
│   ├── extension/          # 浏览器插件
│   └── backend/            # Node.js 服务
├── packages/
│   ├── shared-types/       # 共享 TypeScript 类型 + Zod schema
│   ├── skills/             # Skills 引擎（可独立测试）
│   └── adapters/           # Memos/飞书 适配器
├── docs/
├── pnpm-workspace.yaml
├── tsconfig.base.json
├── .env.example
└── README.md
```

### 命名与代码风格

- **文件名**：kebab-case（`page-sense.ts`）
- **类型名**：PascalCase（`KnowledgeCard`）
- **函数名**：camelCase
- **常量**：UPPER_SNAKE_CASE
- **每个 Skill 一个独立文件**：`packages/skills/src/<skill-name>/index.ts`
- **每个 Adapter 一个目录**：`packages/adapters/src/<target>/index.ts`

### Git 工作流

- 每个 Phase 一个分支：`phase-0-init`、`phase-1-types`…
- Commit message 格式：`[P{phase}-T{task}] <description>`，例：`[P3-T2] implement Distiller skill`
- 每完成一个 Task 一次 commit，不要攒大 commit

### 质量基线

- TypeScript 严格模式：`strict: true`
- 所有对外 API 用 Zod 校验
- 单元测试覆盖率 ≥ 60%（Skills 层 ≥ 80%）
- Lint：ESLint + Prettier（配置见 Phase 0）

```prisma
model User {
  id            String   @id @default(uuid())
  email         String   @unique
  createdAt     DateTime @default(now())
  settings      UserSettings?
  credentials   UserCredential[]
  saveLogs      SaveLog[]
}

model UserSettings {
  id                  String   @id @default(uuid())
  userId              String   @unique
  defaultTemplate     String   @default("auto")
  defaultTarget       String   @default("memos")
  outputLanguage      String   @default("zh-CN")
  enabledSkillsJson   Json     // { styleMatcher: false, ... }
  createdAt           DateTime @default(now())
  updatedAt           DateTime @updatedAt
  user                User     @relation(fields: [userId], references: [id])
}

model UserCredential {
  id            String   @id @default(uuid())
  userId        String
  target        String   // 'memos' | 'feishu' | ...
  encryptedData String   // AES-256 加密后的 JSON
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  user          User     @relation(fields: [userId], references: [id])

model SaveLog {
  id            String   @id @default(uuid())
  userId        String
  cardId        String
  target        String
  status        String   // 'success' | 'failed' | 'retrying'
  responseJson  Json?
  errorMessage  String?
  createdAt     DateTime @default(now())
  user          User     @relation(fields: [userId], references: [id])
  @@index([userId, createdAt])
}
```

## Phase 3：Skills 引擎实现（4-6 天）

### 目标
实现 MVP 所需的 6 个 Skills（不包含 StyleMatcher，Chronicle 简化为本地记录）。这是项目最核心的部分。

### 通用约束

- 所有 Skill 位于 `packages/skills/src/<skill-name>/`
- 每个 Skill 导出一个 `run(input): Promise<Output>` 函数
- Prompt 作为独立文件放在同目录 `prompt.ts`，便于版本管理
- 所有 Skill 支持 `logger` 注入（DI 形式）
- 单元测试用 mock LLM 响应，不调用真实 API

### 任务列表

#### Task 3.1 LLM Client 封装

**目标**：统一的 Claude API 调用层，供所有 Skill 复用。

**执行步骤**：

1. 安装 `@anthropic-ai/sdk`
2. 创建 `packages/skills/src/llm/client.ts`：
   - 导出 `callClaude(params)` 函数
   - 入参：`{ model, systemPrompt, userPrompt, temperature, maxTokens, stream? }`

3. 创建 `packages/skills/src/llm/models.ts`，定义模型常量：
   ```typescript
   export const MODELS = {
     HAIKU: 'claude-haiku-4-5-20251001',

   ```
4. 创建 `packages/skills/src/llm/structured.ts`：支持 JSON 输出约束（使用 tool_use 模式保证结构化）

**产出**：可复用的 LLM 客户端。

**验收**：
- [ ] 真实调用能返回结果（集成测试，打上 `@integration` 标签）
- [ ] 重试逻辑可测试（mock 失败响应）
- [ ] JSON 模式输出格式稳定

#### Task 3.2 Skill 1：PageSense（页面类型识别）

**目标**：实现 PRD §6.3 的 PageSense。

**执行步骤**：

1. 创建目录 `packages/skills/src/page-sense/`
2. `signals.ts` — 实现非 AI 信号提取：
   - `detectByDomain(url)`：根据域名返回候选类型（如 `github.com` → 倾向 doc）
   - `detectByDOM(html)`：统计代码块数量、列表数量、标题层级
   - `detectByKeywords(text)`：检测"步骤"、"教程"、"观点"等关键词
3. `prompt.ts` — LLM 仲裁 prompt，输入包含 title / cleanText 前 2000 字 / signals 结果
4. `index.ts`:
   ```typescript
   export async function run(input: PageSource): Promise<PageSenseOutput> {
     const signals = collectSignals(input);
     const llmResult = await callClaude({
       model: MODELS.HAIKU,
       systemPrompt: SYSTEM_PROMPT,
       userPrompt: buildUserPrompt(input, signals),
       temperature: 0,
       maxTokens: 200,
     });
     return parseResult(llmResult);
   }
   ```
5. 输出类型：
   ```typescript
   interface PageSenseOutput {
     pageType: PageType;
     confidence: number;
     suggestedTemplate: string;
     signals: string[];
   }
   ```

**产出**：PageSense Skill 完整实现。

**验收**：
- [ ] 对 8 种典型页面的测试样本（放在 `__fixtures__/`）分类准确率 ≥ 80%
- [ ] 信心值 < 0.5 时不崩溃
- [ ] 单元测试覆盖信号层（不依赖 LLM）

#### Task 3.4 Skill 3：Distiller（结构化提炼）**【核心】**

**目标**：按 pageType 分支执行结构化提炼，MVP 仅支持 4 种类型。

**执行步骤**：

1. 创建 `packages/skills/src/distiller/`
2. `prompts/` 目录下每种 pageType 一个 prompt 文件：
   - `tutorial.ts`：提炼 summary / prerequisites / steps / warnings
   - `opinion.ts`：提炼 summary / keyPoints / quotes / counterArguments
   - `news.ts`：提炼 summary / whoWhatWhenWhere / keyFacts
   - `doc.ts`：提炼 summary / apiSignature / params / examples
3. 每个 prompt 文件内容结构：
   - System prompt（含 2~3 个 few-shot 示例，作为 prompt cache 的稳定前缀）
   - User prompt 模板函数
   - 输出 JSON schema（用于 tool_use 约束）
4. `index.ts`：
   ```typescript
   export async function run(input: DistillerInput): Promise<DistillerOutput> {
     const promptModule = PROMPT_MAP[input.pageType];
     if (!promptModule) {
       // 降级到 generic
       return runGeneric(input);
     }
     return await callClaude({
       model: MODELS.SONNET,
       systemPrompt: promptModule.systemPrompt,
       userPrompt: promptModule.buildUserPrompt(input),
       temperature: 0.3,
       maxTokens: 2000,
     });
   }
   ```
5. `generic.ts` — 通用降级策略（所有未识别类型回退到此）
6. 实现 **prompt caching**：system prompt 末尾加 `cache_control: { type: 'ephemeral' }`

**产出**：支持 4 种 pageType + 通用降级的 Distiller。

## 附录：默认决策表

遇到 PRD 未明确的决策，优先采用以下默认值：

| 场景 | 默认选择 | 原因 |
|---|---|---|
| 日期时区 | 统一 UTC（ISO 8601 with Z） | 避免时区混乱 |
| 语言默认 | 跟随页面语言，fallback 到 zh-CN | 中文用户为主 |
| 未识别 pageType | 回退到 generic | 保证主流程不中断 |
| LLM 响应超时 | 30 秒 | Claude 合理上限 |
| LLM 重试次数 | 3 次（指数退避） | 业界惯例 |
| 本地卡片上限 | 100 张 | 避免 IndexedDB 膨胀 |
| Memos 默认 visibility | private | 隐私优先 |
| 飞书文档默认位置 | 用户指定的 folder，未指定则根目录 | - |
| 标签数量 | 3~5 个 | PRD §6.3 |
| 错误展示 | 友好文案 + 错误码 | 便于用户反馈但不暴露细节 |
| API 版本 | 全部走 `/api/v1/` | 为 v2 留空间 |
| JWT 有效期 | access 1 小时 / refresh 30 天 | 平衡安全与便利 |
